# OpportuMap PRD

> 작성일: 2026-04-15 | 버전: v1.0

## 1. 배경 및 문제 정의

### 왜 만드는가?
- 공모전/해커톤/예창패/지원금 정보가 콘테스트코리아, 링커리어, 위비티, K-Startup, 올콘, 스펙토리 등 **최소 6개 사이트에 파편화**되어 있음
- 대부분 리스트 나열 형태로, 마감일 기반 시간축 관리가 불가능 — **마감일 놓침이 가장 빈번한 페인포인트**
- 기존 서비스는 "공모전 정보 포털"일 뿐, 사용자의 **지원 여정 전체를 관리하는 도구**가 아님

### 해결하려는 핵심 문제 (1문장)
> 흩어진 공모전·해커톤·지원금 공고를 캘린더 한 곳에 모아, 마감일을 놓치지 않고 원클릭으로 지원까지 연결하는 서비스.

| 지원 프로세스 분절 | 중간 | 각 공고 주최사 사이트 방문 | 공고 확인 → 별도 사이트 이동 → 양식 다운로드 → 제출. 3~4단계 이상 |

### 우리의 차별화 포인트
1. **캘린더 퍼스트**: 리스트가 아닌 캘린더가 메인 화면. 마감일·접수시작일·심사일을 시간축에 배치
2. **올인원 수집**: 공모전 + 해커톤 + 예창패 + 정부지원금 4개 카테고리를 한 곳에 (기존 서비스는 1~2개만 커버)
3. **원클릭 지원 연결**: 공고 상세에서 지원 페이지로 직행 링크 + 필요 서류 체크리스트
4. **웹 서비스**: 슥삭(앱 전용), 모모365(문화 특화)와 달리 웹에서 범용 접근

| 기존 대안 | 콘테스트코리아 + 링커리어 + K-Startup을 번갈아 방문 |

## 5. Job Stories (JTBD)

1. When 여러 공모전에 관심이 있지만 마감일을 자주 놓칠 때, I want to 모든 마감일을 캘린더 한 곳에서 보고 싶다, so I can 중요한 기회를 놓치지 않을 수 있다
2. When 공모전/해커톤/지원금 정보를 찾기 위해 6개 사이트를 돌아다닐 때, I want to 한 곳에서 통합 검색하고 필터링하고 싶다, so I can 탐색 시간을 줄이고 적합한 기회만 빠르게 찾을 수 있다
3. When 관심 있는 공고를 발견했을 때, I want to 바로 지원 페이지로 이동하고 필요 서류를 확인하고 싶다, so I can 지원 프로세스를 빠르게 시작할 수 있다

## 7. 유저 플로우

### 시나리오 1: 기회 탐색 → 캘린더에서 발견
```
[랜딩 페이지] → [캘린더 메인 화면 (이번 달 공고 시각화)] → [날짜 클릭] → [해당일 마감 공고 리스트] → [공고 카드 클릭] → [상세 모달: 요약, 자격, 상금, 지원링크]
```
- 캘린더에서 마감일 밀도를 색상 강도로 표현 (많을수록 진하게)
- 카테고리별 색상 구분 (공모전=파랑, 해커톤=초록, 예창패=주황, 지원금=빨강)

### 시나리오 2: 검색 → 필터 → 지원
```
[검색바에 키워드 입력 "AI 해커톤"] → [결과 리스트 + 필터 사이드바] → [필터 적용 (대학생, 서울)] → [공고 카드 클릭] → [상세 페이지] → ["지원하기" 버튼 → 주최사 지원 페이지로 새탭 이동]
```

## 8. 디자인 방향

- **디자인 도구**: Stitch MCP (1순위)
- **모바일 우선 여부**: 반응형 (데스크톱 캘린더 중심 + 모바일 리스트 뷰 전환)
- **톤/무드**: 클린 & 기능적. 캘린더 앱(Notion Calendar, Google Calendar) 느낌. 과도한 장식 배제
- **컬러 방향**: 카테고리 구분 컬러 4색 + 중성 배경. 보라-핑크 그라데이션 금지
- **DESIGN.md**: 별도 파일로 생성 예정
