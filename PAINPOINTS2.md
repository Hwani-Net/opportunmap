## R1: 신규 방문자 첫인상 (5초 이내 무엇을 해야 하는지 알 수 있는가?)

- **PP-001**: 헤더에 로고 "OpportuMap"만 있고 서비스 설명이 없음 → 처음 방문자는 이름만 보고 공모전 캘린더인지 알 수 없음 → 헤더 로고 아래 또는 옆에 한 줄 태그라인 추가 ("공모전·해커톤·지원금 통합 캘린더")
- **PP-002**: Hero 영역이 사실상 없음. h1 "공모전·해커톤·지원금, 한 곳에서"가 작은 텍스트(text-2xl)로 필터바 위에 조용히 위치 → 가치 제안이 시각적으로 눈에 안 띔 → Hero 섹션을 시각적으로 강조하거나 히어로 배너 영역 별도 구성
- **PP-003**: 최초 진입 시 CTA(콜투액션)가 없음. "지원하기", "공모전 보기" 등 명확한 행동 유도 없이 바로 캘린더/리스트로 시작 → 신규 방문자가 무엇을 눌러야 할지 혼란 → 상단에 주요 CTA 버튼 배치
- **PP-004**: 캘린더 뷰가 기본값(데스크톱)인데, 빈 달이면 아무것도 안 보임 → 데이터 로딩 전 캘린더만 덩그러니 표시됨 → 로딩 상태를 리스트 뷰처럼 skeleton으로 표시 (현재 캘린더는 스피너만)
- **PP-005**: 통계 카드 3개(진행 중 N건, 이번 주 마감 N건, 새로 등록 N건)가 컨텐츠 **아래**에 위치 → 핵심 수치인데 스크롤해야 보임 → 상단으로 이동하거나 헤더 근처에 배치

## R2: 모바일 반응형 (breakpoint, 모바일 전용 UI)

- **PP-006**: FilterBar의 드롭다운 3개(분야/대상/지역)가 모바일에서 가로 스크롤 없이 줄바꿈됨 (`flex-wrap items-center justify-between`) → 좁은 화면에서 레이아웃이 2~3줄로 늘어나 필터 영역이 지나치게 넓어짐 → 모바일에서 필터를 drawer/bottom sheet로 이동
- **PP-007**: 카테고리 버튼 4개(공모전/해커톤/예창패/지원금)가 `overflow-x-auto flex-nowrap`이지만, 모바일에서 스크롤 가능 여부가 시각적으로 표시되지 않음 → 사용자가 더 버튼이 있는지 모름 → 스크롤 힌트(그라데이션 페이드) 추가
- **PP-008**: 헤더의 모바일 검색창이 버튼 탭으로 토글되는데, 검색창이 열릴 때 레이아웃 shift 발생 → 검색 아이콘 클릭 → 헤더 아래에 갑자기 SearchBar 등장 → 부드러운 애니메이션 또는 검색창 상시 표시
- **PP-009**: `isMobile` 감지를 `window.innerWidth <= 768`로 초기 렌더 시 1회만 계산 (`useFilters.ts:25`) → 화면 크기 변경(resize) 시 뷰 모드가 자동 전환되지 않음 → `useEffect` + `resize` 이벤트로 동적 감지 필요
- **PP-010**: ListView의 그리드가 `grid gap-3` (1열)로 고정 → 태블릿(md:)에서 2열, 데스크톱(lg:)에서 3열 스켈레톤은 있지만 실제 ListView는 항상 1열 → ContestCard가 큰 화면에서 공간 낭비 (스켈레톤과 실제 레이아웃 불일치)

- **PP-015**: `showPast` 체크박스 레이블이 "마감 포함"으로 너무 짧음 → 의미가 모호 (마감된 공모전을 포함한다는 뜻인지 불분명) → "마감된 공고 보기"로 변경

## R4: 캘린더 상호작용 (이벤트 클릭, 월 이동, 주간 뷰)

- **PP-016**: `dayMaxEvents={3}` 설정으로 4개 이상이면 "+N more" 표시되는데, 이 링크를 클릭하면 FullCalendar 기본 팝오버가 뜸 → `body.detail-modal-open .fc-popover { display: none }` CSS가 있어 모달 열린 상태에서는 팝오버가 숨겨지지만, 모달 닫힌 상태에서는 팝오버가 스타일 일관성 없이 표시됨 → 커스텀 팝오버 또는 dayClick으로 해당 날짜 리스트 모달 표시
- **PP-017**: listWeek(주간) 뷰에서 EventChip 컴포넌트가 어떻게 렌더링되는지 별도 처리 없음 → `eventContent={(arg) => <EventChip event={arg} />}` 단일 렌더러가 월간/주간 양쪽에 적용됨 → 주간 리스트 뷰에서 이벤트 표시가 어색할 수 있음
- **PP-018**: 캘린더에서 이벤트를 클릭하면 `detail` URL 파라미터로 상세 모달이 열리는데, 브라우저 뒤로가기 시 모달이 닫히고 이전 캘린더 상태로 돌아감 → 기술적으로는 올바르나, 뒤로가기로 모달 닫기가 가능한지 사용자가 모름 → "ESC로 닫기" 힌트 추가 (이미 키보드 Escape 지원 있음)
- **PP-019**: 캘린더 `height="auto"` + `min-height: 120px` per cell → 공고가 많은 달에 캘린더가 매우 길어질 수 있음 → 최대 높이 제한 + 스크롤 옵션 필요
- **PP-020**: 캘린더 뷰에서 북마크 기능 없음 → 리스트 뷰에서는 각 카드에 북마크 버튼이 있지만, 캘린더 이벤트 칩(EventChip)에는 북마크 액션 없음 → 상세 모달에서는 북마크 버튼이 없음 (ContestDetailModal에 북마크 버튼 누락)

## R5: 상세 모달 완성도 (정보 누락, 체크리스트, 접근성)

- **PP-021**: `ContestDetailModal`에 북마크 버튼 없음 → 모달에서 바로 북마크 불가, 리스트 뷰로 돌아가야 함 → 모달 상단 닫기 버튼 옆에 북마크 버튼 추가
- **PP-022**: `contest.field` (분야) 정보가 상세 모달에 없음 → 리스트 카드에는 태그로 표시되지만 모달 2×2 그리드에는 빠져 있음 → 분야 정보 추가
- **PP-023**: 체크리스트 초기화 기능 없음 → `localStorage`에 저장되어 모달 닫고 다시 열어도 유지되지만, 전부 해제하는 "초기화" 버튼 없음 → 체크리스트 헤더에 "모두 해제" 버튼 추가
- **PP-024**: 체크리스트가 비어 있으면(`checklist.length === 0`) 아무것도 표시 안 됨 → "필요 서류" 섹션 자체가 사라짐 → "등록된 서류 정보가 없습니다" 등 안내 문구 표시
- **PP-025**: 모달 스크롤 시 헤더(제목/닫기 버튼)가 고정되지 않음 → 긴 내용에서 스크롤 내리면 닫기 버튼이 보이지 않음 → 모달 헤더 sticky 처리
- **PP-026**: `contest.applyUrl`이 비어 있을 때 "지원하기 →" 버튼이 빈 href(`""`)로 렌더링될 수 있음 → `{contest.applyUrl && ...}` 조건 없이 항상 렌더링 (ContestDetailModal:209) → URL 없으면 버튼 비활성화 또는 숨김 처리
- **PP-027**: 마감일까지 남은 일수(D-day)가 상세 모달에 없음 → 리스트 카드에는 D-day 뱃지 있지만 모달에는 마감 기간만 날짜로 표시 → 모달 상단에 D-day 뱃지 추가

## R6: 소스코드 버그/개선점

- **PP-028**: `useFilters.ts:25` `isMobile` 계산이 SSR/첫 렌더 시 window 접근으로 런타임 에러 가능성 → `typeof window !== "undefined"` 가드가 있지만 hydration mismatch 위험 → `useEffect` 내에서 감지하도록 리팩터 권장
- **PP-029**: `filterContests`에서 `filters.search` 검색이 `title`과 `organizer`만 대상 → `summary`, `field`, `category` 레이블 검색 불가 → 검색 범위 확장 또는 검색 대상 명시
- **PP-030**: `ContestDetailModal`의 `checked` 초기 상태가 `contest.checklist.length`로 결정되는데, 이후 같은 모달 재사용 시(`key={selectedContest?.id ?? "closed"}`) contest가 바뀌어도 `useState` 초기값이 재계산되지 않음 → key 기반 remount로 해결되어 있으나, key가 `"closed"`로 공유되면 이전 contest의 `checked` 상태가 잠깐 보일 수 있음
- **PP-031**: `getContestById` 함수가 `lib/firestore`에서 import되지만, 실제로는 메모리 배열에서 find하는 단순 유틸 → 이름이 Firestore 직접 조회처럼 오해를 유발 → 함수명을 `findContestById`로 변경 권장
- **PP-032**: `subscribeContests` 실시간 구독을 사용하지만 에러 시 재구독 로직 없음 → Firestore 연결 끊김 시 빈 화면 유지, 자동 복구 안 됨 → 에러 상태에서 retry 버튼 외에 자동 재시도 로직 필요
- **PP-033**: `contest.createdAt`이 optional(`createdAt?: Timestamp`) → "새로 등록" 통계 카드에서 `createdAt`이 없는 공고는 0으로 처리 → 실제 새 공고가 카운트에서 빠질 수 있음
- **PP-034**: `ListView`의 그리드가 `grid gap-3` 단일 열 → 스켈레톤은 `md:grid-cols-2 lg:grid-cols-3`인데 실제 리스트는 1열 → 시각적 불일치 및 공간 낭비

## R7: 접근성/a11y

- **PP-035**: `ContestCard`에 `role="button"`이 있지만 `article` 태그를 사용 → `article`에 `role="button"`은 시맨틱 충돌 → `div` 또는 `button` 태그로 변경
- **PP-036**: 모달 열릴 때 focus가 모달로 이동하지 않음 → 키보드 사용자가 ESC로 닫을 수 있지만, 모달 열린 직후 포커스가 배경에 남아 있음 → `useEffect`에서 모달 열릴 때 닫기 버튼 또는 모달 컨테이너로 `focus()` 이동
- **PP-037**: 모달이 열려 있을 때 배경 콘텐츠에 포커스 트랩 없음 → 키보드 Tab으로 모달 밖 요소에 접근 가능 → `focus-trap` 라이브러리 또는 `inert` 속성으로 배경 비활성화
- **PP-038**: FilterBar의 카테고리 버튼들에 `aria-pressed` 속성 없음 → 스크린 리더가 토글 상태를 알 수 없음 → `aria-pressed={isActive}` 추가
- **PP-039**: `select` 요소에 `aria-label`은 있으나 현재 선택값이 스크린 리더에 제대로 전달되는지 확인 불가 → 네이티브 `select`는 기본적으로 괜찮으나, 커스텀 디자인이 되면 접근성 깨짐 주의
- **PP-040**: `index.html`의 `og:image` 없음 → 소셜 공유 시 이미지 없이 텍스트만 표시 → OG 이미지 추가 필요 (R9와 연관)

## R8: 성능/로딩

- **PP-041**: `CalendarView`가 `lazy` import지만 `Suspense` fallback이 단순 스피너 → 캘린더 초기 로딩(FullCalendar 번들)이 느릴 수 있는데 진행률 피드백 없음 → 스피너 대신 캘린더 형태의 skeleton 표시
- **PP-042**: FullCalendar 관련 패키지(`@fullcalendar/react`, `daygrid`, `interaction`, `listPlugin`) 4개가 번들에 포함 → 용량이 상당함에도 tree-shaking이 제한적 → 번들 분석 후 필요 플러그인만 유지 검토
- **PP-043**: `useContests`가 `subscribeContests`로 실시간 구독을 하는데, 탭 비활성 시에도 구독 유지 → 불필요한 Firestore read 발생 → `document.visibilitychange` 이벤트로 탭 비활성 시 구독 일시 중단 고려
- **PP-044**: 폰트가 Google Fonts CDN 2회 요청 (`index.html`과 `index.css` 양쪽에서 동일 URL import) → 중복 네트워크 요청으로 초기 로딩 지연 → 한 곳에서만 import
- **PP-045**: 통계 카드(진행 중, 이번 주 마감, 새로 등록)가 `contests` 전체 배열 기반으로 매 렌더마다 재계산됨 → `useMemo` 없이 직접 `.filter()` 3회 호출 → `useMemo`로 최적화

- **PP-046**: `og:image` 메타태그 누락 → SNS/카카오톡 공유 시 이미지 없이 썸네일 빈칸 → OG 이미지(1200×630) 생성 및 추가

- **PP-049**: `canonical` URL이 `https://fuel-price-alert-kr2.web.app/`로 하드코딩 → 실제 서비스 도메인 변경 시 SEO 오류 → 환경변수 기반 동적 생성 또는 실제 도메인으로 업데이트
- **PP-050**: 구조화 데이터(JSON-LD) 없음 → 검색엔진이 공모전 이벤트를 구조화 데이터로 인식 불가 → `Event` 스키마 또는 `ItemList` 스키마 추가 고려

## R10: 추가 페인포인트 (R1-R9에서 누락된 항목)

- **PP-051**: 북마크가 `localStorage` 기반 → 로그인 사용자라도 기기 간 북마크 동기화 안 됨 → Firebase Auth가 구현되어 있으므로 로그인 시 Firestore에 북마크 저장
- **PP-052**: "마감됨" 공고가 기본적으로 필터링되어 숨겨지는데(`showPast: false` 기본값), 이미 지원한 공고를 다시 확인하려면 "마감 포함" 체크박스를 찾아야 함 → 북마크한 마감 공고는 자동 포함하는 옵션 고려
- **PP-053**: 공고 정렬 기능 없음 → 기본 순서(Firestore 입력 순)로만 표시 → 마감일 임박순, 최신 등록순, 상금 높은순 정렬 옵션 필요
- **PP-054**: 알림 기능 없음 → 마감 임박 공모전에 대한 푸시/이메일 알림 기능 없음 → 서비스 핵심 가치("마감일 놓치지 마세요")와 불일치 → 알림 구독 기능 필요
- **PP-055**: 공고 수가 많아졌을 때 무한 스크롤 또는 페이지네이션 없음 → 전체 데이터를 한 번에 Firestore에서 구독 → 100개+ 공고 시 성능 저하 + 비용 증가
- **PP-056**: Footer 컴포넌트가 import되어 있으나 내용 미확인 → 저작권, 문의, 개인정보처리방침 등 법적 필수 항목 포함 여부 확인 필요
- **PP-057**: 공모전 공유 버튼 없음 → 개별 공모전을 카카오톡/클립보드로 공유하는 기능 없음 → 딥링크 URL 복사 버튼 추가
